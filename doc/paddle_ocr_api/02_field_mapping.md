# 02 Field Mapping

## Core Principle

When mapping, only ask one thing:

- Which layer of `document.v1` should this Paddle field land in

Currently allowed placement layers:

1. Core structure layer: `type/sub_type/bbox/text/lines/segments/tags/derived`
2. Common trace layer: `metadata` that multiple providers may share
3. Provider raw trace layer: Paddle private fields, retained in `metadata/source`

## Top-Level Mapping

| Paddle Field | `document.v1` Field | Notes |
| --- | --- | --- |
| Provider fixed value | `source.provider` | Currently fixed to `paddle` |
| Input file path | `source.raw_files.source_json` | Injected by adapter outer layer |
| Page count | `page_count` | Determined by the number of pages |

## Page Mapping

| Paddle Field | `document.v1` Field | Notes |
| --- | --- | --- |
| `dataInfo.pages[i].width` | `pages[i].width` | Preferred |
| `dataInfo.pages[i].height` | `pages[i].height` | Preferred |
| `prunedResult.width` | `pages[i].width` | Fallback |
| `prunedResult.height` | `pages[i].height` | Fallback |
| Page index | `pages[i].page_index` | Starting from 0 |
| Fixed value | `pages[i].unit` | Currently fixed to `pt` |

## Block Mapping

| Paddle Field | `document.v1` Field | Notes |
| --- | --- | --- |
| `block_bbox` | `bbox` | Normalized bbox |
| `block_content` | `text` | Normalized text |
| `block_label` | `type/sub_type/tags` | Goes through `block_labels.py` |
| Line/segment split result | `lines/segments` | Goes through `content_extract.py` |
| `block_id` | `source.raw_block_id` | Preserves original source |
| `block_label` | `source.raw_type` | Preserves original type |
| `block_bbox` | `source.raw_bbox` | Preserves original bbox |
| `block_content[:200]` | `source.raw_text_excerpt` | For troubleshooting |
| Original path | `source.raw_path` | Points to original JSON path |

Implemented mapping examples:

| `block_label` | `type` | `sub_type` | `tags` |
| --- | --- | --- | --- |

## `derived` Mapping

Currently `derived` is mainly generated by provider rules, see:

- `backend/scripts/services/document_schema/provider_adapters/paddle/trace.py`

Typical rules:

- `doc_title -> derived.role = title`
- `abstract -> derived.role = abstract`
- `reference_content -> derived.role = reference_entry`
- `formula_number -> derived.role = formula_number`
- `header/footer -> derived.role = header/footer`

## Do NOT Do This

1. Do not directly stuff Paddle private fields as new main contract fields.
2. Do not re-interpret `block_label` in the translation layer.
3. Do not temporarily change `type/sub_type` semantics for a single fixture.
